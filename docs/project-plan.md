# Mac Note — 프로젝트 기획서

> 마지막 업데이트: 2026-07-11

---

## 1. 프로젝트 개요

### 컨셉
macOS 스타일의 **자유 배치 메모 웹 앱**.  
브라우저 위에서 동작하는 데스크톱 앱 경험을 목표로 한다.  
메모를 캔버스 위에 자유롭게 배치하고, 창처럼 이동·크기 조절·레이어 정렬이 가능하다.

### 핵심 방향
- **로컬 우선** : 서버 없이 localStorage로 동작. 계정 없이 즉시 사용.
- **미니멀 UI** : 불필요한 버튼을 숨기고 필요할 때만 노출하는 인터페이스.
- **macOS 감성** : 창 컨트롤(빨강·노랑·초록), 블러 패널, 반투명 UI.

## 3. 데이터 스키마 (v1.0 확정)

### 3-1. bxArr 메모 객체 구조
```javascript
{
  // 고유 식별자 & 레이아웃
  id: number,              // timestamp
  zindex: number,          // 레이어 순서
  width: number,           // px 단위
  height: number,          // px 단위
  top: string,             // "30px" 형식
  left: string,            // "30px" 형식
  status: 'response'|'fullsize', // 윈도우 상태
  
  // 메타데이터
  name: string,            // 제목 (빈 문자열 = 제목 없음)
  createdAt: number,       // timestamp
  updatedAt: number,       // timestamp
  
  // 상태 플래그
  favorite: boolean = false,  // 즐겨찾기
  alwaysTop: boolean = false, // 최상위 고정
  locked: boolean = false,    // 위치 고정 (드래그 비활성화)
}
```

### 3-2. localStorage 텍스트 콘텐츠 (ID를 key로)
```javascript
{
  textContent: string,  // 메모 본문 텍스트
}
```

### 3-4. 배경 이미지 설정 스키마 (v0.2 예정)
```javascript
// localStorage
bgMode: 'theme' | 'library' | 'upload' | 'link'
bgValue: string
// - theme: 'light' | 'dark' | 'auto'
// - library: assets 내 이미지 경로
// - upload: IndexedDB key (예: 'bg-upload-1720670000')
// - link: 절대 URL

bgMeta: {
  source: 'library' | 'upload' | 'link',
  updatedAt: number,
  name?: string,
  size?: number,
}

// IndexedDB (업로드 파일 저장)
db: 'macnote-assets'
store: 'backgrounds'
record: {
  id: string,
  blob: Blob,
  mime: string,
  createdAt: number,
}
```

## 4. 데이터 마이그레이션 전략

### 4-1. 현재 문제
기존 localStorage 메모들이 새 필드(`favorite`, `alwaysTop`, `locked`)가 없어 `undefined` 상태.

### 4-2. 마이그레이션 로직
- 페이지 로드 시 `loadBox()` → `bxArr` 복원 전 마이그레이션 함수 실행
- 부족한 필드는 기본값으로 채움
- 기존 불필요한 필드(`link`, `task`)는 유지 (호환성)

### 4-3. 마이그레이션 함수 위치
`js/box.js`의 `loadBox()` 함수 내부에 통합.

### P2 — 단기 개선 (v0.2 계획)
#### 5-1. Undo/Redo 기능 (우선순위 최고)
**기능 설명**: Ctrl+Z (취소) / Ctrl+Y (다시 실행) 단축키로 작업 되돌리기

**Undo/Redo 대상**
- 메모 생성
- 메모 삭제
- 메모 이동 (드래그)
- 메모 리사이징
- 메모 제목 변경
- 메모 상태 플래그 (즐겨찾기, 최상위 고정, 위치 고정)

**텍스트 변경 정책 (정책 B)**
- 포커스를 잃었을 때만 텍스트 변경을 한 번의 작업으로 기록
- 입력 중엔 Undo 불가 (효율성), 포커스 잃은 후 이전 상태로 복원 가능

**UI 피드백**
- 위치: 화면 우측 하단
- 스타일: 어두운 배경 + 흰 텍스트 + ⓘ 정보 아이콘
- 메시지 예: "실행 취소 - 메모 생성", "다시 실행 - 메모 이동"
- 자동 소멸: 1.5~2초 후 사라짐

**기술 스펙**
- 단축키: Ctrl+Z (Undo) / Ctrl+Y (Redo)
- 스택 크기: 최근 50개까지만 유지
- 저장소: 변경 로그 기록 형식 `{type, memoId, before, after}`

#### 5-2. 메모 세팅 패널 반응형 개선
**요구사항**: 메모 가로 너비가 375px 이하일 때 세팅 패널이 메모 박스 전체를 덮도록 표시

**구현 예정**:
- JavaScript 브레이크포인트로 375px 감지
- 좁을 때: `.bx-set`이 `position: fixed` + 전체 화면 덮음
- 넓을 때: 현재 상태 유지 (우측 40%)
- 세팅 항목 스택화 (세로 정렬)
  - 제목 입력칸 full-width
  - 토글 항목들 (레이블 좌측, 스위치 우측)

#### 5-3. 데이터 구조 정리
- 구 데이터 마이그레이션 함수 테스트 (v0.1.4에서 추가됨)
- localStorage 데이터 버전 관리 기능 추가 (향후 스키마 변경 시 대비)

#### 5-4. UI 완성도
- 헤더 제목 오버플로우 처리 (✅ 완료)
- 상태 플래그 헤더 아이콘 표시 (예정: 🔒 위치고정, ⬆️ 최상위, ⭐ 즐겨찾기)

#### 5-5. 기타
- `top_calender.js` 제거 또는 캘린더 패널 구현 (v2.0 이상)

#### 5-6. 배경 이미지 설정 기능 (v0.2)
기존 테마 배경 외에 사용자 배경 지정 기능 추가.

선택 옵션:
